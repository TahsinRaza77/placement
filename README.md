# Admission Chance Predictor

A small Flask web app that predicts a graduate applicant's "Chance of Admit"
using a Linear Regression model trained on `Admission_Predict.csv`.

## Project structure

```
admission_app/
├── Admission_Predict.csv   # dataset
├── train_model.py          # trains + saves the model (run once)
├── app.py                  # Flask server
├── requirements.txt
├── model/                  # generated by train_model.py
│   ├── model.pkl
│   ├── scaler.pkl
│   └── meta.pkl
├── templates/
│   └── index.html
└── static/
    ├── style.css
    └── script.js
```

## Setup

```bash
pip install -r requirements.txt
```

## 1. Train the model (creates model/*.pkl)

```bash
python train_model.py
```

This reads `Admission_Predict.csv`, cleans it, scales the 7 input features
with `StandardScaler`, fits a `LinearRegression` model, and pickles the
scaler + model into the `model/` folder. Re-run this any time the dataset
changes.

## 2. Run the web app

```bash
python app.py
```

Then open **http://127.0.0.1:5000** in your browser.

## How it works

- The form collects the same 7 features your original script used:
  GRE Score, TOEFL Score, University Rating, SOP, LOR, CGPA, and Research.
- On submit, the page sends a JSON POST to `/predict`.
- The Flask route validates the input, scales it with the saved
  `StandardScaler`, runs it through the saved `LinearRegression` model,
  and returns the predicted chance of admit as a percentage.
- The result is shown as an animated seal/gauge with a short verdict.

## Notes on changes from the original script

- Removed the `ydata_profiling` report generation, hardcoded Windows file
  path, and one-off `test1` prediction — none of that is needed for the
  live web app, and profiling is a one-time EDA step, not part of the
  request/response cycle.
- The model is trained **once** via `train_model.py` and loaded from disk
  at server startup, rather than retrained on every request.
- Added input validation and graceful error handling in `/predict`.
